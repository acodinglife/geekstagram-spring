What you need

@Controller

Gives a servlet like behaviour managed by Spring (simplified way of work) 


							@Controller
							public class HomeController {
								/* Methods and members */
							}

@RequestMapping

Associate a pattern url with a HTTP verb with a method 


							@RequestMapping(value = "/signup", method = RequestMethod.GET)
							public String signup() {
								return "signup";

							}

When sending a GET /signup, the method is called and send the "signup" view located in src/main/resources/templates

@PathVariable("nameOfValue")

Get the variable(s) in path 


							@RequestMapping("/posts/{username}")
							public String postByUserName(@PathVariable("username") String username){
								/* render the view needed*/
							}

@RequestParam("value")

Get the request parameter (query or body, dependending on the verb) 


							@RequestMapping(value = "/", method = RequestMethod.POST)
							public String login(@RequestParam("email") String mail, @RequestParam("password") String password){
								/* render the view needed*/
							}

Render variables

Use the "Model" class 


							@RequestMapping("/signup")
							public String signup(Model model) {
								model.addAttribute("error","This is an error");
								return "signup";
							}
